# Changelog — 2026-07-19

## feat: implement dispenser comparison grid widget with mobile slider, dynamic categories and webflow integration

### Descripción
Se desarrolló e implementó el nuevo widget interactivo **Sistemas de Dispensación Comparativa** (Grid de Comparación) basado en las especificaciones de diseño de Figma. Admite carga dinámica a través de la API del CMS de Webflow, cambio de imágenes principales al hacer clic sobre miniaturas, navegación mediante deslizamiento horizontal (*swipe*) con dots indicadores en dispositivos móviles, y filtrado dinámico de secciones mediante nombres de categorías y descripciones provistos desde el CMS de Webflow.

### Archivos creados / modificados
- **`widgets/sistemas-dispensacion-comparativa/sistemas-dispensacion-comparativa.js`** — Script principal del widget con lógica de inyección de estilos CSS, cargado dinámico/fallback, mapeo de miniaturas variantes, efectos de transiciones de imagen, y dots navegacionales del slider móvil.
- **`widgets/sistemas-dispensacion-comparativa/preview.html`** — Panel de previsualización local para pruebas de carga dinámica por CMS simulado y fallback estático con filtros de categorías.
- **`widgets/sistemas-dispensacion-comparativa/README.md`** — Documentación técnica de integración, nombres de campos CMS requeridos, asignación de clases CSS en Webflow y código embed HTML.

### Cambios detallados
- **Layout de Figma:** Cuadrícula de 4 columnas en desktop con líneas divisorias verticales, tipografía Montserrat y Rubik, bordes y colores slate (#506D85), verde (#4aa25d) y azul (#48a9c5) de acuerdo al diseño.
- **Galería Interactiva:** Al hacer clic en las miniaturas de productos compatibles, la imagen principal se reemplaza suavemente usando una animación de cross-fade (200ms de opacidad) y el elemento activo se resalta con un borde de color destacado.
- **Filtrado por Categorías (Recomendado por Atributo Embed):** Se determinó y documentó como enfoque principal el filtrado mediante el atributo `data-category-filter` (ej. `data-category-filter="Soportes"`) en los contenedores Embed del widget. Esto permite al usuario tener **una única lista de colección de Webflow oculta global** con todos los productos de todas las categorías en la página, y que el script distribuya y filtre de forma limpia los elementos en sus respectivas secciones, evitando conflictos de múltiples consultas repetidas de CMS y problemas de auto-detección global.
- **Detección Automática de CMS:** Se restableció y mejoró la rutina de auto-detección basada en el elemento `.jypesa-disp-name`. Si por algún motivo la clase contenedora principal `.jypesa-sdc-cms-source` no se colocó o fue removida, el widget rastreará los nodos de Webflow de forma jerárquica (localmente en el padre y luego globalmente en la página) para ubicar los productos de Webflow reales y evitar saltar prematuramente al fallback estático.
- **Slider Responsivo Móvil:** En resoluciones de pantalla iguales o menores a 768px, el grid se adapta automáticamente a un carrusel de desplazamiento horizontal que muestra un producto por slide. Adiciona puntos navegacionales dinámicos en la base que se sincronizan de forma bidireccional con el scroll.
- **Ocultamiento Inteligente de Variantes:** Si el producto no tiene imágenes de marcas/botellas compatibles adicionales en el CMS, la fila de miniaturas se oculta por completo de forma limpia para evitar contenedores vacíos o imágenes rotas. Adicionalmente se agregaron validaciones avanzadas para descartar imágenes con `src` vacío, URLs autorreferenciales del navegador (que resuelven la ruta de la página actual) y placeholders de carga asíncrona tipo Data URIs (`data:image/svg+xml;base64,...`).
- **Color de Badge Personalizable:** Se añadió el campo `.jypesa-disp-badge-color` para permitir modificar el color de fondo de las etiquetas (badges) directamente desde el CMS (ej. `#4aa25d` para Economy, `#48a9c5` para Premium, etc.).
- **Ocultamiento Inteligente de Acciones y Extracción Flexible de Enlaces:** Si los enlaces de la Guía de Instalación o la Ficha Técnica están vacíos o no asignados en el CMS, los botones correspondientes no se renderizan en absoluto. Se rediseñó la función `getLink` para leer de forma híbrida: si la clase está en un contenedor `div`, busca un elemento `a` en su interior para extraer la URL; si el elemento tiene la URL escrita directamente como texto plano (común en bloques de texto vinculados al CMS en Webflow), lee su contenido de texto para habilitar la descarga del archivo PDF.
